# JT1078Http2RTP
1078上级平台看视频是通过下级平台提供的HTTP连接查看的，常规做法是针对这个HTTP写一个播放器。对于既直接接入设备又接入下级平台的需要开发两套播放器。

此项目提供另一种思路，将HTTP请求收到的数据模拟成设备再发送给1078视频平台，可以只用一套播放逻辑即可。

# 此项目可和RTVS配合实现同时接入设备和下级平台

# 接口地址

启动JTH2R_Web后 通过下面接口可启动一个HTTP转1078RTP连接
```
http://localhost:5000/Api?httpUrl=[1078HTTP请求地址]&Server1078=[转发视频服务器地址]&Port1078=[转发视频服务器端口]
```

# 流程



视频控件--->视频服务器---->网关

网关判断如果是来源于下级平台，通过809发起视频请求，等待下级平台返回HTTP请求信息后调用JT1078Http2RTP接口即可。


网关---->JT1078Http2RTP---->视频服务器---->视频控件
